# 要求事項
## アーキテクチャ
- クリーンアーキテクチャとドメイン駆動設計を採用する
- package by featureを原則とするディレクトリ構成
- 共有モジュールは `features/shared/` に配置する
- フロントエンドはReact Nativeを使用する
- バックエンドはHonoを使用する

## データフロー
- アプリを起動すると自動で録音が開始される
- 音声データをバックエンド（Hono）に送信
- HonoがGemini APIへ文字起こしをリクエスト
- Honoが結果を受け取りフロントエンドに返す
